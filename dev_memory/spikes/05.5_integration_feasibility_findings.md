# Spike 05.5 - Integration Feasibility Mock Spike Findings

Status: in_progress

This document is the findings output for ROADMAP phase `05.5`. The spike is
mock-only and must stay isolated from production `src/agent/` code.

## Question

Can the automated decision core (candidate engine + constraint layer +
experience memory + exploration schedule) reproduce or exceed the already
proven human semi-auto LLM tuning workflow with low human intervention?

This spike does not test whether LLM-guided compiler-option tuning has value;
that has already been observed in the user's manual workflow. It tests whether
the automated loop can:

- converge in 30-50 mock rounds,
- beat naive baselines,
- learn second-order interactions,
- recover from poor LLM proposals, bad experience, noise, duplicates, and
  crash/resume.

## Scope

Included:

- Synthetic objective with known optimum requiring second-order interaction.
- Minimal `OptionV0` IR.
- Four strategies: random, LLM-only, local mutation, full agent.
- Deterministic mock LLM with good/poor quality knobs.
- Minimal constraint layer, memory, schedule, and crash/resume simulation.

Excluded:

- Real gbs.
- Subprocess or process management.
- Workspace protection and spec mutation.
- LangGraph.
- Real LLM API calls.
- Production `src/agent/` changes.

## Implementation Log

### 2026-05-30 - Subtask 05.5.1 Objective / IR / Random Runner

Implemented:

- `OptionV0` and default option catalog.
- `SyntheticObjective` with exact known optimum
  `{-O3, -funroll-loops, -fA, -fB}`.
- Second-order trap: `-fA` or `-fB` alone is harmful; together they add +12.
- Extra-option penalty so random supersets cannot tie the known optimum.
- `RandomStrategy` and seeded `run_strategy()` baseline runner.

### 2026-05-30 - Subtask 05.5.2 MockLLM / LLMOnly / LocalMutation

Implemented:

- `MockLLM` with `quality="good"` and `quality="poor"`.
- `LLMOnlyStrategy`, which trusts mock LLM output without constraints or memory.
- `LocalMutationStrategy`, a one-flip hill climber around the best successful
  combo.
- Duplicate-rate helpers on `RunResult`.

Validation:

- `LocalMutationStrategy` reaches the local optimum `{-O3, -funroll-loops}` on
  the noiseless objective.
- It evaluates `{-O3, -funroll-loops, -fA}` and
  `{-O3, -funroll-loops, -fB}` as worse neighbors.
- It never proposes the `-fA`/`-fB` pair, so the second-order-interaction
  baseline is now meaningful for later full-agent comparison.
- `spikes/05.5_integration_feasibility/tests`: 14 passed.
- Production `tests/`: 451 passed.

## Exit Criteria

- Full agent finds top-5% optimum within 30-50 rounds in at least 80% of seeds.
- Duplicate trial rate is below 5%.
- Known always-failing combos are never actually executed.
- Injected bad hard rule is corrected by Canary downgrade or does not
  permanently block the optimum.
- Crash/resume preserves round, candidate history, and experience state.
- Full agent clearly outperforms random and LLM-only baselines.
- Full agent finds the second-order-interaction optimum while local mutation
  remains stuck in a local optimum.
