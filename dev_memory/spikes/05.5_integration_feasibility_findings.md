# Spike 05.5 - Integration Feasibility Mock Spike Findings

Status: done

This document is the findings output for ROADMAP phase `05.5`. The spike is
mock-only and must stay isolated from production `src/agent/` code.

## Question

Can the automated decision core (candidate engine + constraint layer +
experience memory + exploration schedule) reproduce or exceed the already
proven human semi-auto LLM tuning workflow with low human intervention?

This spike does not test whether LLM-guided compiler-option tuning has value;
that has already been observed in the user's manual workflow. It tests whether
the automated loop can:

- converge in 30-50 mock rounds,
- beat naive baselines,
- learn second-order interactions,
- recover from poor LLM proposals, bad experience, noise, duplicates, and
  crash/resume.

## Scope

Included:

- Synthetic objective with known optimum requiring second-order interaction.
- Minimal `OptionV0` IR.
- Four strategies: random, LLM-only, local mutation, full agent.
- Deterministic mock LLM with good/poor quality knobs.
- Minimal constraint layer, memory, schedule, and crash/resume simulation.

Excluded:

- Real gbs.
- Subprocess or process management.
- Workspace protection and spec mutation.
- LangGraph.
- Real LLM API calls.
- Production `src/agent/` changes.

## Core Finding

The decision loop integration is viable, but the spike exposed the central
Phase 07 risk: **noise-robust second-order interaction discovery**.

In the noiseless objective, near-miss guided interaction is the independent
driver of second-order discovery:

| Objective | guided interaction | random fallback | Top-5% hits / 20 seeds | Exact optimum hits / 20 seeds |
|---|---:|---:|---:|---:|
| no noise | on | on | 20 | 20 |
| no noise | on | off | 20 | 20 |
| no noise | off | on | 0 | 0 |
| no noise | off | off | 0 | 0 |

With default benchmark noise (`noise_sigma=2.0`), the same 2x2 ablation loses
mechanistic separation. All four configurations reach top-5% in 11/20 seeds
and exact optimum in 4/20 seeds:

| Objective | guided interaction | random fallback | Top-5% hits / 20 seeds | Exact optimum hits / 20 seeds |
|---|---:|---:|---:|---:|
| noisy | on | on | 11 | 4 |
| noisy | on | off | 11 | 4 |
| noisy | off | on | 11 | 4 |
| noisy | off | off | 11 | 4 |

Root cause: the near-miss signal is deliberately narrow. The useful single
options `-fA` and `-fB` have deterministic score drop 1.0 from the local
optimum, and the current suspect window is `[0.75, 1.25]`. Default noise has
sigma 2.0, so a one-shot measurement often pushes useful options out of the
near-miss window or makes unrelated observations look better than they are.

This is not a failure of the mock loop plumbing; it is the exact technical risk
Phase 07 must handle. Noise-robust interaction discovery likely needs one or
both of:

- repeated evaluation of the same combo with mean/median aggregation to reduce
  effective noise;
- statistically significant near-miss detection using confidence intervals
  or bootstrap tests.

Both directions depend on Phase 08 statistical-significance machinery. The
handoff is explicit: Phase 7.0 should expand from pure constraint-solver
performance into candidate-search strategy and noise-robust interaction
discovery, with Phase 08 providing the statistical tools.

## Strategy Comparison

Scenario split matters. Good LLM can directly propose the optimum, so the
honest comparison there is trial efficiency rather than best score. Poor LLM
cannot reliably propose the interaction, so the comparison there is robustness
and best-score recovery. Noisy top-5% hit rates are limited by the core finding
above.

| Scenario | Strategy | Seeds | Top-5% hits | Median best score | Median duplicate rate | Notes |
|---|---|---:|---:|---:|---:|---|

## Exit Criteria

- Full agent finds top-5% optimum within 30-50 rounds in at least 80% of seeds.
- Duplicate trial rate is below 5%.
- Known always-failing combos are never actually executed.
- Injected bad hard rule is corrected by Canary downgrade or does not
  permanently block the optimum.
- Crash/resume preserves round, candidate history, and experience state.
- Full agent clearly outperforms random and LLM-only baselines.
- Full agent finds the second-order-interaction optimum while local mutation
  remains stuck in a local optimum.

## Conclusions

The spike produced a useful split result:

- Integration plumbing is viable. The synthetic loop, option IR, mock LLM,
  constraint layer, memory, duplicate filtering, false-positive suspicion
  counter, and crash/resume reconstruction all work in isolation.
- Good LLM comparison should be efficiency-based. LLM-only can reach the
  optimum because good LLM can propose it directly, but full agent avoids the
  repeated trial budget burn.
- Poor LLM comparison should be robustness-based. Full agent clearly improves
  over poor LLM-only on noisy median best score, while rejecting conflicts,
  unknown flags, and duplicates before spending budget.
- Noiseless guided interaction works: the 2x2 ablation proves guided
  interaction is the independent second-order driver when measurement noise is
  absent.
- Noisy guided interaction is not solved. With `noise_sigma=2.0`, the
  near-miss signal is drowned out; all 2x2 configurations show the same noisy
  top-5% hit behavior. Treat this as the top candidate-engine risk, not as a
  closed problem.

## Recommendations For Phase 07

- Phase 7.0 should expand from "constraint solver performance" into
  "candidate search strategy + noise-robust interaction discovery".
- Phase 07 candidate generation should not rely on exhaustive pair or
  fixed-size combo enumeration except as a bounded diagnostic fallback.
- Phase 08 statistical-significance tools should feed candidate generation:
  repeated evaluations, mean/median aggregation, confidence intervals, and
  bootstrap tests are likely needed before near-miss signals are reliable.
- Phase 07 should preserve the parts that did work:
  constraint prefiltering, duplicate rejection without trial budget, explicit
  rejection reasons, suspicion/canary-style false-positive recovery, and
  resume from canonical history.
- The Phase 07 acceptance criteria should require an ablation similar to this
  spike: disable random/brute fallbacks and show the intended search mechanism
  still contributes independently.
