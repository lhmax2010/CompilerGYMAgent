# Spike 05.5 - Integration Feasibility Mock Spike Findings

Status: in_progress

This document is the findings output for ROADMAP phase `05.5`. The spike is
mock-only and must stay isolated from production `src/agent/` code.

## Question

Can the automated decision core (candidate engine + constraint layer +
experience memory + exploration schedule) reproduce or exceed the already
proven human semi-auto LLM tuning workflow with low human intervention?

This spike does not test whether LLM-guided compiler-option tuning has value;
that has already been observed in the user's manual workflow. It tests whether
the automated loop can:

- converge in 30-50 mock rounds,
- beat naive baselines,
- learn second-order interactions,
- recover from poor LLM proposals, bad experience, noise, duplicates, and
  crash/resume.

## Scope

Included:

- Synthetic objective with known optimum requiring second-order interaction.
- Minimal `OptionV0` IR.
- Four strategies: random, LLM-only, local mutation, full agent.
- Deterministic mock LLM with good/poor quality knobs.
- Minimal constraint layer, memory, schedule, and crash/resume simulation.

Excluded:

- Real gbs.
- Subprocess or process management.
- Workspace protection and spec mutation.
- LangGraph.
- Real LLM API calls.
- Production `src/agent/` changes.

## Implementation Log

### 2026-05-30 - Subtask 05.5.1 Objective / IR / Random Runner

Implemented:

- `OptionV0` and default option catalog.
- `SyntheticObjective` with exact known optimum
  `{-O3, -funroll-loops, -fA, -fB}`.
- Second-order trap: `-fA` or `-fB` alone is harmful; together they add +12.
- Extra-option penalty so random supersets cannot tie the known optimum.
- `RandomStrategy` and seeded `run_strategy()` baseline runner.

Validation:

- `spikes/05.5_integration_feasibility/tests`: 7 passed.
- Production `tests/`: 451 passed.

Review:

- Verdict: Approve.
- Fairness audit passed: unique optimum, effective second-order trap,
  non-tied supersets, plausible random baseline chance, and clean spike
  isolation.
- Follow-up: re-check the `-fA`/`-fB` single-option penalty when
  `LocalMutationStrategy` lands.

### 2026-05-30 - Subtask 05.5.2 MockLLM / LLMOnly / LocalMutation

Implemented:

- `MockLLM` with `quality="good"` and `quality="poor"`.
- `LLMOnlyStrategy`, which trusts mock LLM output without constraints or memory.
- `LocalMutationStrategy`, a one-flip hill climber around the best successful
  combo.
- Duplicate-rate helpers on `RunResult`.

Validation:

- `LocalMutationStrategy` reaches the local optimum `{-O3, -funroll-loops}` on
  the noiseless objective.
- It evaluates `{-O3, -funroll-loops, -fA}` and
  `{-O3, -funroll-loops, -fB}` as worse neighbors.
- It never proposes the `-fA`/`-fB` pair, so the second-order-interaction
  baseline is now meaningful for later full-agent comparison.
- `spikes/05.5_integration_feasibility/tests`: 14 passed.
- Production `tests/`: 451 passed.

Review:

- Verdict: Approve.
- Local mutation was independently verified to be a stable local-optimum
  baseline: repeated seeds did not find the `-fA`/`-fB` pair.
- Follow-up for full-agent reporting: because good-quality `MockLLM` can
  propose the exact optimum, comparisons against LLM-only must be split by
  scenario. Good LLM should emphasize trial efficiency; poor LLM should
  emphasize fallback robustness and best-score recovery.

### 2026-05-30 - Subtask 05.5.3 FullAgentStrategy Core

Implemented:

- `ExperienceMemory` derived from prior `TrialOutcome` history.
- `ConstraintLayer` for duplicate, unknown-option, conflict,
  known-failed-subset, and soft-blocked candidate rejection.
- Suspicion counter that forces a soft-blocked combo after repeated rejection.
- `FullAgentStrategy` schedule:
  warmup -> mock LLM -> generic pair-jump exploration -> local mutation ->
  random fallback -> deterministic enumeration.

Validation:

- Poor LLM conflicts and unknown options are rejected before execution:
  actual compile failures are 0 in the full-agent run.
- Full agent reaches the second-order optimum in 10/10 noiseless poor-LLM
  seeds, while poor LLM-only misses it and local mutation stays at 108.
- Good LLM-only can also reach the optimum, so the comparison is efficiency:
  full agent duplicate rate 0.0 vs LLM-only duplicate rate >= 0.85 in the probe.
- `spikes/05.5_integration_feasibility/tests`: 21 passed.
- Production `tests/`: 451 passed.

Review:

- Verdict: Approve with follow-up.
- Ablation exposed the key caveat:
  - full agent as implemented: finds the second-order optimum in repeated seeds;
  - pair-jump disabled: still finds the optimum;
  - random/enumerated fallback disabled: no longer finds the optimum.
- Interpretation: the current result is driven by deterministic enumeration
  fallback over the small toy option space, not by scalable interaction
  discovery. This is a useful spike finding, but it means the "learns
  second-order interactions" exit criterion is not honestly satisfied yet.
- Decision: take follow-up path (a). The next subtask should remove or sharply
  limit brute enumeration and make success depend on interaction exploration
  with a plausible real-scale Phase 07 path.

### 2026-05-30 - Subtask 05.5.4 Guided Interaction Follow-up

Implemented:

- Removed deterministic `_enumerated_candidates`.
- Replaced unguided pair-jump with near-miss guided interaction exploration.
- Added `ExperienceMemory.near_miss_additions()`:
  after the strategy observes single additions around the current best combo,
  mildly-worse additions become bounded interaction suspects.
- `FullAgentStrategy` now proposes pairs only from those observed suspects.

Validation:

- Poor LLM, random fallback disabled:
  full agent finds the second-order optimum in 20/20 seeds.
- Poor LLM, random fallback disabled, guided interaction disabled:
  0/20 optimum hits and the candidate stream exhausts.
- Therefore the second-order result is no longer attributable to deterministic
  enumeration over the toy option space.
- `spikes/05.5_integration_feasibility/tests`: 23 passed.
- Production `tests/`: 451 passed.

## Exit Criteria

- Full agent finds top-5% optimum within 30-50 rounds in at least 80% of seeds.
- Duplicate trial rate is below 5%.
- Known always-failing combos are never actually executed.
- Injected bad hard rule is corrected by Canary downgrade or does not
  permanently block the optimum.
- Crash/resume preserves round, candidate history, and experience state.
- Full agent clearly outperforms random and LLM-only baselines.
- Full agent finds the second-order-interaction optimum while local mutation
  remains stuck in a local optimum.
