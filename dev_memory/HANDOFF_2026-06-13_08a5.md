## 08a.5 Delivered Scope

Implemented in `8936849`:

- `StatisticalResult` schema for single-comparison statistics.
- `compare_run_records()` as the side-effect-free baseline-vs-candidate entry point.
- `comparison_scope="single_comparison"` and `adjusted_for_multiple_testing=false`.
- `significant_single_comparison` boolean; no bare `significant`.
- Direction-aware signed effect:
  - higher-is-better: candidate - baseline,
  - lower-is-better: baseline - candidate.
- `relative_effect_pct=None` when baseline mean is effectively zero.
- Paired comparison by matching `pair_key`, preserving baseline pair order.
- Paired difference sequence still runs autocorrelation and ESS diagnostics.
- Unpaired high-autocorrelation comparisons are inconclusive.
- Verdict gates:
  - `n_valid < 5` or `ESS < 3` -> `inconclusive`,
  - `5 <= n_valid < 10` or `3 <= ESS < 5` -> low-power `inconclusive`,
  - small-n autocorrelated paired data -> low-power `inconclusive` per Med-1,
  - adequately powered CI excluding zero -> significant improvement/regression,
  - adequately powered CI including zero -> `no_difference`.
- No multiple-comparison correction, adaptive rerun action, outlier policy, or
  candidate engine.

Python 3.10 compatibility follow-ups:

- `0c87bb3`: lower project runtime to Python `>=3.10`; replace 3.11-only
  `datetime.UTC`, `typing.Self`, and `tomllib` assumptions.
- `b78c744`: add `tests/__init__.py` so absolute `tests.fixtures` imports work
  on Ubuntu/Python 3.10.
