Phase 02 / Subtask 2.4 - SoT discovery helpers

Requirements:
- REQUIREMENTS.md section 4.2.4
- REQUIREMENTS.md section 4.1.4

Code changes:
- Added `TrialLoadError` and `TrialDiscoveryError` for trial YAML read/discovery failures.
- Added alias-free `TrialYamlLoader` and a 1 MiB `MAX_TRIAL_RECORD_BYTES` cap.
- Added `DiscoveredTrialRecord` and `TrialStartupValidationInputs` dataclasses.
- Added `load_trial_record(path)`:
  - rejects symlink paths
  - rejects missing, oversized, non-UTF-8, empty, non-mapping, alias-bearing, unsafe-tag, and schema-invalid YAML
  - requires the immutable trial `integrity` block
  - verifies the payload hash before returning `TrialRecord`
- Added `load_trial_record_for_layout(layout, path)`:
  - enforces that record namespace matches `NamespaceLayout`
  - enforces that path equals `trials/data/YYYY-MM/trial_<trial_id>.yaml`
- Added `iter_trial_record_paths(layout)` and `discover_trial_records(layout)` for deterministic canonical SoT scanning.
- Added `collect_trial_startup_validation_inputs(layout, compiler_type=...)` and `existing_trial_compiler_versions(...)` for section 4.1.4 startup compatibility inputs.
- Exported the new public APIs from `agent.__init__`.
