Phase 06 / Subtask 6.4 - Workspace lock probe + LockStatus unknown

Implemented read-only workspace lock hardening for trace cleanup.

Changes:
- Added LockProbeResult and WorkspaceLock.probe_lock().
- probe_lock performs a nonblocking flock probe on the existing run.lock file
  and immediately unlocks if the probe succeeds.
- Kept WorkspaceLock._write_holder unchanged; run.lock is never
  atomic-replaced and holder metadata is still written through the held fd.
- Updated trace cleanup lock classification so the kernel flock decides
  free vs busy, while holder metadata only explains the holder.
- Added LockStatus="unknown" for unreadable holder metadata and stale metadata
  while the real flock is busy.
- Updated tests to use real active locks for held_by_self / held_by_other
  instead of metadata-only simulation.
