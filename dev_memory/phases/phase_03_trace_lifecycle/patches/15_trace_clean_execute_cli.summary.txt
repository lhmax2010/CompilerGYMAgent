Subtask 3.11 - trace clean execute and CLI

Implemented:
- Added CleanResult, CleanExecutionRefusedError, StaleCleanPlanError, and execute_clean_plan().
- execute_clean_plan() trusts CleanPlan execution predicates and does not recompute session/checkpoint/time protection.
- Execution acquires the workspace lock for normal plans and supports force-clean-inactive-only when the current process already owns the lock.
- Execution rejects stale plans under the lock when trace file size or validated line count changed after planning.
- Atomic trace rewrite uses precomputed removable byte ranges, a same-directory temporary file, fsync, os.replace(), and parent-directory fsync.
- Default backup writes original trace bytes to _trash/<UTC timestamp>/events.jsonl; backup=False and --no-backup skip it.
- Added agent console script with:
  - agent clean trace
  - agent clean trace --yes
  - agent clean trace --force-clean-inactive-only
  - agent clean trace --no-backup
  - agent doctor trace
- clean trace defaults to dry-run. doctor trace is read-only.
