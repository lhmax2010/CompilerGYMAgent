Subtask 05.5.4 - Guided Interaction Follow-up

Addressed the 05.5.3 review finding that full-agent second-order discovery was
attributable to deterministic enumeration over the toy option space.

Implemented:

- Removed deterministic size-1..4 enumeration fallback from FullAgentStrategy.
- Replaced unguided pair-jump with near-miss guided interaction exploration.
- Added ExperienceMemory.near_miss_additions():
  - finds single-option additions around the current best combo
  - keeps additions whose score drop is small
  - limits the suspect set before forming pairs
- FullAgentStrategy now proposes interaction pairs from observed near-miss
  suspects rather than enumerating the whole catalog.
- Added tests proving:
  - full agent finds the second-order optimum with poor LLM even when random
    fallback is disabled
  - disabling guided interaction under the same conditions prevents discovery
  - deterministic enumeration fallback is no longer present

Validation:

- .venv/bin/python -m pytest spikes/05.5_integration_feasibility/tests -q -> 23 passed
- .venv/bin/python -m pytest tests/ -q -> 451 passed

Ablation:

- FullAgentStrategy, poor LLM, random fallback disabled: 20/20 optimum hits
- Guided interaction disabled, poor LLM, random fallback disabled: 0/20
  optimum hits and candidate stream exhaustion
